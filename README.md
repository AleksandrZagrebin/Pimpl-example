# cpp-pimpl-example

Пример использования идиомы PImpl (Pointer to Implementation) в C++.

---

## О проекте

Проект демонстрирует применение идиомы PImpl на примере класса `User`.  
Цель — показать, как скрыть детали реализации, ускорить компиляцию и сохранить бинарную совместимость (ABI).

---

## Что такое ABI

**ABI** (Application Binary Interface) — это интерфейс на уровне скомпилированного кода.  
Он определяет:

- размер и выравнивание типов
- расположение полей в классе
- порядок передачи аргументов в функции
- имена функций в бинарном файле (name mangling)

ABI важен, когда библиотека (`.so`, `.dll`) обновляется без перекомпиляции программы, которая её использует.

---

## ABI vs API

| | API | ABI |
|---|---|---|
| **Уровень** | Исходный код | Бинарный код |
| **Когда важен** | На этапе компиляции | На этапе выполнения |
| **Что определяет** | Сигнатуры функций, доступные методы | Размеры типов, смещения полей |
| **Изменение** | Требует перекомпиляции пользователя | Может сломать совместимость |
